## 1. 제품 개요

**한 줄 정의** 아이의 만세력(사주)과 성향검사를 종합해 학업·미래 직업 방향을 제시하고 부모의 추가 질문에 답하는 모바일 웹 앱.

**목적** ① 아이의 타고난 결과 지금의 모습을 이해 ② 그에 맞춰 현재 무엇을 가꿀지 진단.

**타깃** 유아~초등(핵심) 자녀를 둔 부모. 초등 고학년+는 아이 자기응답 병행.

**플랫폼** 모바일 웹(원핸드, 최대폭 440px). 단일 페이지 앱.

**포지셔닝/톤** "재미로 보는 성장 나침반". 따뜻·희망적·개인화. 시그니처 = 아이를 '별자리'로 표현하는 새벽하늘 컨셉.

**불변 원칙 (전 화면 관통)**

- **참고용·비확정**: 운명·성적·미래를 확정하지 않는다. 항상 '경향·가능성'.
- **IQ 수치·우열 금지**: 지능은 강점 성향으로만. 진짜 지능검사(K-WISC)는 전문가 영역 → 필요 시 안내로 연결.
- **강점 기반·비낙인**: 단점은 '함께 키울 지점'. '피해야 할 직업' 류 금지.
- **삼각측량**: 사주(선천) + 검사(후천 강점·흥미) + 부모 관찰. 단일 소스로 결론 금지.
- **연령 게이트**: 나이로 응답주체(부모/아이)·문항 세트·직업 출력 형태 분기.

## 4. 유저 플로우

### 4.1 메인 플로우

```
[S0 인트로/동의]
   │  동의 체크 → CTA 활성
   ├──"샘플 아이로 체험"──────────────┐
   ▼                                  │
[S1 기본정보]                          │
   │  이름·성별·양음력·생년월일·시각    │
   │  (생년월일 → 나이 자동 → 연령게이트)│
   │  검증 OK                          │
   ▼                                  │
[S2 문항: 핵심]                        │
   │  한 화면 한 질문, 탭→자동 진행     │
   │  (적응형 분기: 트리거시 심화 삽입)  │
   │  연령≥10 → 흥미 6문항             │
   │  (선택) "더 깊이 보기" → 심화 모듈  │
   ▼                                  │
[S3 DEEP 분석] ◄───────────────────────┘
   │  1) 결정론 엔진: 만세력·점수·매핑
   │  2) AI 서사: 리포트 JSON 생성
   │  (실패 시 재시도 CTA)
   ▼
[S4 결과 리포트] ⇄ [S5 Q&A]

```

### 4.2 적응형 분기 (S2 내부)

```
핵심 응답 수집 중
  ├ NS≥4 ─────▶ 삽입: 억제/안전 상황 문항
  ├ HA≥4 ─────▶ 삽입: 자율성/대처 문항
  ├ ER1≤2 ────▶ 삽입: 정서표현 문항
  ├ MI 상위 동률 ─▶ 삽입: 강제선택 쌍(tie-break)
  └ P↔GRIT 불일치 ▶ 삽입: 흥미주도 확인 문항
심화 모듈(opt-in): 기질2차·성격2차·EF전환·정서2차·가치랭킹·개방형관찰

```

### 4.3 엣지/에러

- 시각 '모름' → 시주·대운 제외 계산, 리포트에 신뢰도 낮춤 표기.
- 문항 중 '잘 모르겠어요' → 해당 신호 결측, 가중치 재분배.
- AI 호출 실패 → 리포트 화면에 오류 카드 + "다시 분석" CTA(결정론 결과는 보존).
- 뒤로가기: S2 첫 문항에서 → S1. S1 → S0.

### S2 · 성향 문항

**목적** 구성개념별 신호 수집. 한 화면 한 질문, 탭 후 자동 진행. **공통** 상단바(뒤로·진행바·N/총), 섹션 칩(색상), 세리프 질문, 이름 삽입.

```

│ 몰입하는 놀이는?      │     │ 새로운 것에 먼저      │
│ ┌──────────────────┐ │     │ 다가가 살펴봐요       │
│ │🎨 그림·만들기     │ │     │ [전혀][별로][보통]…  │
│ ├──────────────────┤ │     │  ○○○○○ (5점)      │
│ │🧩 퍼즐·숫자·규칙  │ │     └──────────────────────┘
│ ├──────────────────┤ │
│ │📖 이야기·역할놀이 │ │     [다중선택 그리드]
│ └──────────────────┘ │     ┌──────────────────────┐
└──────────────────────┘     │ 잘하는 것 모두 골라요 │
                             │ [🎨][🧩][📖][🎵]     │
```

### S3 · DEEP 분석 로딩

**목적** 결정론 엔진 계산 + AI 서사 대기 동안 신뢰감 있는 연출(체감 정밀도).

```
┌──────────────────────────────┐
│                              │
│         ( ◜ ✦ ◞ )           │  회전 궤도(2링)+빛나는 코어
│                              │
│   서아의 미래를 그리는 중     │
│   DEEP 분석 중…              │
│   ● 사주·오행·대운 계산       │  단계 리스트
│   ● 강점·흥미·기질 점수화     │  (완료=●, 진행=◍, 대기=○)
│   ◍ 학업·직업 매핑           │
│   ○ 미래 시나리오 서사화      │
└──────────────────────────────┘

```

**동작** 단계 애니메이션은 실제 파이프라인(엔진→AI)과 병행. AI 응답 도착 시 전 단계 완료 후 리포트로 전환. 실패 시 리포트 화면 오류 카드.

```
│ 🤍 함께 키워갈 지점 ◦×2      │ C
│ 📚 학업 성취                 │ D
│   강점교과 [수학][과학][국어] │   칩
```

### S5 · Q&amp;A 채팅 (S4 하단 내장)

**목적** 리포트 기반 후속 질문 응답.

```
│ 💬 더 궁금한 점을 물어보세요  │
│ [강점 키우기][약한 부분 걱정] │  추천 칩
│ [지금 활동?][직업 왜 어울려?] │
│ ┌── 대화 로그 ──────────────┐│
│ │        (내 질문) ▟        ││  me 버블(우)
│ │ ▛ (AI 답변 2~4문장)       ││  ai 버블(좌), 로딩 dots
│ └────────────────────────────┘│
│ [ 입력____________________ ][↑]│  sticky 입력

```

**동작** 칩 탭 = 즉시 전송. AI 컨텍스트 = 계산 결과 + 생성 리포트 + 최근 히스토리. 답변 2~4문장, 필요 시 '참고용' 부드럽게 상기.

## 6. 기능 스펙 (모듈별)

### F1 · 만세력 엔진 (결정론)

**책임** 생년월일시 → 사주 4기둥·오행 분포·일간·십성·신강신약·용신 근사·대운. **규칙**

- **일주**: JDN 기반, 앵커 1900-01-01 = 갑술(60갑자 index 10). (검증됨: 2000-01-01=무오)
- **연주**: 입춘(≈2/4) 경계로 사주연도 보정. `(연-4) mod 10/12`.
- **월주**: 절기 근사표로 월지 → 오호둔으로 월간.
- **시주**: 시진(2시간) → 시지, 오서둔으로 시간. 시각 모름 시 생략.
- **오행 분포**: 8글자(시 모름 시 6) 오행 카운트.
- **십성(간이)**: 일간 오행 기준 생/극 관계로 비겁·식상·재성·관성·인성 집계.
- **신강/신약(간이)**: 일간+인성(일간을 생하는 오행) 비율 ≥0.45 → 신강.
- **용신 근사**: 신강→식상·재성·관성 / 신약→비겁·인성.
- **대운**: 순행/역행 = (양년&amp;남 or 음년&amp;여)→순행. 월주 기준 간지 시퀀스. 시작나이 **근사**. **정밀도/백로그** 절기·진태양시·서머타임 정밀 보정, 음→양력 변환(윤달), 대운수(절기거리/3) = **KASI 기준 검증 후 정밀화**. 현재는 근사임을 UI 명시.

### F2 · 문항 뱅크 &amp; 응답 수집 (v0.2)

**책임** 구성개념별 문항 제공, 연령·적응형·티어에 따른 노출, 신호 태그 집계. **구성개념** 기질4(T)·성격3(C)·자기조절/실행기능(EF)·정서조절(ER)·다중지능8(MI)·학습/동기(LS/MO)·그릿(GRIT)·가치(VAL)·대인(SOC)·흥미(RIASEC)·개방형(OBS). **형식** 5점 빈도 / 단일선택 / 다중선택 그리드 / 강제선택 쌍 / 랭킹 / 슬라이더 / 개방형. **티어** 핵심(필수, 완결 리포트 가능) + 심화(opt-in) + 적응형 삽입. **품질** 역채점 문항으로 일관성 점검, '모름' 허용, 읽기수준 초6~중1.

> 문항 전문·태그·적응형 트리거는 [**question-bank-v0.2.md**](http://question-bank-v0.2.md)가 원천(source of truth).

### F3 · 점수·매핑 엔진 (결정론)

**책임** 신호 → 직업 클러스터(12)·강점 교과(8) 점수 산출. **절차**

1. 신호 정규화: 오행·십성·MI·흥미(RIASEC) 각 벡터.
2. 클러스터 점수 `base = wMI·MI + wINT·흥미 + wWX·오행 + wSS·십성`, `final = base × 기질적합계수(0.85~1.15)`.
  - 가중치: 흥미 수집 `MI .32 / INT .27 / WX .24 / SS .17`, 미수집 `MI .45 / WX .35 / SS .20`.
3. 상위 2개 선정 + **부족 오행 계열 1개 강제 편입**(보완), 계열 편중 시 다양성 규칙.
4. 교과 점수 `0.6·MI→교과 + 0.4·오행→교과` → 상위 2~3.
5. 각 상위 클러스터의 **기여 신호(contrib)** 리스트 산출(리포트 근거 칩).

> 매핑 테이블·가중치·환경태그·용신 규칙 전체는 [**academic-career-mapping-logic.md**](http://academic-career-mapping-logic.md)가 원천.

### F4 · 조합 해석 규칙

**책임** 척도 간 조합으로 해상도 향상(단순 합산 넘어서). 결과를 리포트/처방에 주입. **핵심 규칙(예)** HA×자율성(적응/위축), 자율성+연대감(안정 지표), 자극추구×실행기능(충동/안전), 인내력×그릿(신뢰도), 연대감×사회적민감성(대인직업), 자기초월(가치·자연·예술), 정서조절 저(학업 전 정서 코칭 우선), 가치 1순위(직업 tie-break).

> 규칙 전체는 [**question-bank-v0.2.md](http://question-bank-v0.2.md) §5**가 원천. 구현: 우선 AI 프롬프트 주입 → 이후 엔진 코드화(백로그).

### F5 · AI 서사 레이어

**책임** 결정론 결과를 받아 **한국어 리포트 JSON** 생성. 계산은 하지 않음(서사만). **입력** child, 사주 요약, 십성, 기질/성격/EF/ER 레벨, 학습/동기/대인, **계산된 강점교과·topClusters(jobs·contrib)**, 대운 흐름(순풍/살펴봄), 부족오행, 개방형 관찰. **제약(시스템 프롬프트)**

- 한국어·부모에게 다정하게. 운명·성적·IQ·건강 확정 금지, 가능성·경향으로. 지능=강점 성향.
- **직업·교과는 제공된 세트 안에서만**, 각 '왜'는 제공된 contrib 근거. 새로 지어내지 않음.
- 단점=함께 키울 지점. 미래=방향 2~3개. **출력 계약** → §8 리포트 스키마. **Q&amp;A** 시스템 프롬프트 + (계산 결과+리포트 컨텍스트) + 최근 히스토리, 2~4문장.

### F6 · Q&amp;A

**책임** 리포트 맥락 유지 대화. 추천 질문 칩 제공. 결측/실패 시 재시도 안내. 자기이해·양육 참고 톤 유지, 필요 시 전문상담 안내.

## 9. 기술 아키텍처

**프론트엔드** 단일 페이지 모바일 웹. 상태는 인메모리(브라우저 스토리지 미사용 원칙, 세션 종료 시 소멸 → 개인정보 유리). 화면 라우팅은 클라이언트.

**AI 연동** Anthropic Messages API. 리포트=1회 호출(JSON 강제), Q&amp;A=대화형.

- **개발/데모**: 아티팩트 샌드박스 내 키 없는 프록시 호출.
- **운영**: 서버리스 **백엔드 프록시**에 실제 API 키 보관, 프론트는 프록시만 호출(키 노출 금지). 레이트리밋·로깅·PII 최소화.

**모듈 경계(구현 파일 제안)** manseryeok(F1) · questionBank(F2) · scoring(F3) · combinationRules(F4) · aiClient+prompts(F5) · ui(화면) · designTokens. 결정론 모듈은 순수함수로 테스트 가능하게.

**결정론 우선** 점수·매핑·조합은 코드가, 서사만 AI가. AI 실패해도 결정론 결과는 유지·표시.
